# RFC — Requests for Comments and Design Proposals

Design process for major decisions. RFC = Request for Comments.

## RFC process

1. **Propose** — Create rfc/NNNN-title.md in Draft status
2. **Review** — Request feedback from leads (1-2 weeks)
3. **Iterate** — Respond to feedback, revise
4. **Accept** — Architecture Lead approves or rejects
5. **Implement** — Code follows RFC spec
6. **Finalize** — Mark status Accepted; update docs and code

## RFC status
- **Draft** — Initial proposal, gathering feedback
- **Review** — Under active review
- **Accepted** — Ready to implement
- **Implemented** — Code exists and matches RFC
- **Deferred** — Postponed to future release
- **Withdrawn** — Author or reviewers decided against
- **Superseded** — Replaced by newer RFC

## RFCs in this project
- 0001-name-and-scope.md — Original COGANT naming and scope
- 0002-ir-schemas.md — Intermediate representation schema design

## What goes in an RFC

- Problem statement
- Proposed solution with examples
- Alternatives considered and rejected
- Implementation plan
- Testing strategy
- Backward compatibility concerns
- Open questions and risks
