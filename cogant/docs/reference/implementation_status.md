### Type-checking gate — bound

The `mypy --strict` figure in `METRICS.yaml` should be read as *errors among reported diagnostics*, not a completeness certificate. Verified state: `0` genuine errors are emitted for the implementation `.py` sources under both the canonical `uv run mypy --strict py/cogant/` and the `uv run --with pydantic …` runs; every residual diagnostic is the pydantic `.pyi` stub-resolution artifact above. The unknown that cannot be excluded in the current toolchain is *masked* consumer-side diagnostics (Any-typed pydantic models flowing into call sites that `--strict` does not flag without `--disallow-any-unimported`). Closing that requires resolving the `.pyi`/pydantic path under the canonical invocation, tracked toward zero.
