## Executive Summary

COGANT provides a reproducible pipeline for turning supported Python code into
GNN-style Active Inference artifacts, then synthesizing a Python package from
that emitted GNN and measuring what survives a second forward pass.

The current evidence supports a **scoped artifact-generation and role-preservation
claim**. It does not support strict graph isomorphism, semantic equivalence for
arbitrary programs, or calibrated behavioral modeling.

## Evidence Chain

The primary measured path is:

```text
source repository
-> ingest/static/normalize/graph
-> semantic-role translation
-> state-space and GNN package generation
-> reverse package synthesis
-> second forward COGANT bundle
-> roundtrip metric comparison
```

The native ledger records per-target role counts, graph size, generated-code
status, `role_preservation_score`, and `roundtrip_status`. `METRICS.yaml`
aggregates that ledger into the headline numbers above.

## What The Result Means

`ROLE_PRESERVED` means the semantic-role population survives the roundtrip at
the configured public threshold. It is a symmetric per-role multiset-overlap
measure, not a recall estimate: extra synthesized roles and dropped original
roles both reduce the score.

It does not imply:

- identical program graphs;
- identical A/B/C/D matrix values;
- identical emitted GNN sections;
- executable semantic equivalence between source and synthesized package; or
- probability calibration against runtime behavior.

Those stronger properties are tracked separately. The current strict
structural-isomorphism count is 0/24.

## Release Claim

Use this wording:

> COGANT maps supported Python repositories into GNN-style Active Inference
> artifacts and preserves semantic-role populations on 24 of 24 native
> roundtrip fixtures. Strict structural roundtrip fidelity is not yet achieved.

Avoid wording that says or implies:

- "strictly isomorphic";
- "semantically equivalent";
- "validated on arbitrary real-world code";
- "calibrated generative model"; or
- "reference-quality runtime behavior."
