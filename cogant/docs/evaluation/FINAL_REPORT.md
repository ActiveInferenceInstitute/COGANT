## Key Architectural Decisions & Lessons

### 1. Galois connection framing
The forward (code→GNN) and reverse (GNN→code) pipelines form an
an approximate forward/reverse relationship. `s_role = role_preservation_score`
is a multiset similarity on semantic roles, while strict structural equivalence is
checked separately. **Lesson**: do not claim bijection when the implemented contract
is role preservation plus an invariant ledger.

### 2. Ontology annotation as fallback classifier
Parser prefix heuristics (s_f*, o_m*, u_c*) fail for non-standard variable
names (s_hidden, o_sensor, u_action). Extended `_parse_ontology_annotation()`
to use `ActInfOntologyAnnotation` concept labels (HiddenState/Observation/Action)
as a fallback pass after the state-space block.
**Lesson**: run prefix heuristics first, ontology second, never skip.

### 3. MatrixFunctions public/private split
`AgentRuntime.__init__` reads `.A/.B/.C/.D` (public attrs). `MatrixFunctions`
originally only had `._A/._B/._C/._D` (private). Fix: expose both.
**Lesson**: check the consuming API's attribute access pattern before designing the provider.

### 4. CONSTRAINT synthesizer naming bug (wave 14, highest-leverage fix)
Planner emitted constraint stubs with `cnst_` prefix. Forward pipeline's
`PreferenceRule` detects CONSTRAINT roles by checking for `check_`, `test_`,
`assert_`, `validate` in function names. `cnst_` matched none of these →
every constraint stub was silently dropped → CONSTRAINT count always 0 in
synthesized package → role-preservation collapse for CONSTRAINT-heavy repos.
**Fix**: rename `cnst_*` → `check_*` in synthesizer output.
**Impact**: 5 repos jumped from DRIFT/FAILED to ROLE_PRESERVED. Overall
ROLE_PRESERVED rate: 61% → 83%.
**Lesson**: naming conventions in the synthesizer must match pattern-matching
conventions in the forward pipeline. These are two ends of the same invariant.

### 5. Default model_name must be lowercase
`ReverseGNNModel.model_name` default was `"CogantModel"`. Property law7 tests
that `model_name == model_name.lower()` always. When raw_name='##' (all special
chars), the GNN section header `## ModelName\n##` causes the parser to see an
empty body → fallback to default → uppercase default → law7 fails.
**Fix**: default = `"cogant_model"`.

### 6. Test from cogant/ subdir, not git root
Running `pytest` from the git root (`projects_in_progress/cogant/`) picks up
`../../evaluation/eval_repos/*/tests/` which require repo-specific dependencies (hypothesis,
etc.). Always run from `cogant/` with `testpaths = ["tests"]` in pyproject.toml.

### 7. Dulwich scaling cliff
dulwich: 8601 nodes / 15441 edges = 1.80 edges/node (vs ~1.10 for all other
repos). Result: 380s / 8.5 GB peak RSS vs <50s / <1.5 GB for all others.
Suspected root cause: edge-driven stage (statespace compiler or graph merge)
with super-linear complexity on edge count. Wave 15 profiling in progress.
**Lesson**: edge density is the primary scaling signal; monitor e/n ratio.

### 8. Parallel multi-agent waves
6–10 simultaneous agents per wave. Conflict avoidance: map file-level
ownership before launch (which files each agent touches). Only block when
actual file path overlap exists. Coverage agents commit frequently, creating
merge windows — always `git add -u` before commit to pick up the right subset.
