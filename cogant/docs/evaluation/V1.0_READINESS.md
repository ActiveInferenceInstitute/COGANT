# COGANT v1.0 Readiness Assessment

Current version: v0.6.0

> **Numeric source of truth:** [`evaluation/METRICS.yaml`](https://github.com/ActiveInferenceInstitute/COGANT/blob/main/evaluation/METRICS.yaml), generated by `tools/regenerate_metrics.py`.

## Verdict

COGANT is ready for a **scoped v1.0 artifact-generator claim**:

> COGANT maps supported Python repositories into GNN-style Active Inference
> artifacts, can synthesize a Python package from the emitted GNN, and preserves
> semantic-role populations for all shipped native roundtrip fixtures.

The claim must remain scoped to the shipped fixtures and supported Python
front-end. It should not be described as arbitrary-program semantic equivalence,
broad strict graph isomorphism, calibrated behavioral modeling, or held-out
generalization.

## Roundtrip Interpretation

The current native ledger reports:

- targets: 25
- role-preserved rows: 25
- drift rows: 0
- failed rows: 0
- strict structural-isomorphism rows: 1
- mean role-preservation score: 1.0000
- score source: `current_native`

`ROLE_PRESERVED` means the semantic-role multiset survives the
forward -> reverse -> forward loop at the public threshold. It does not mean the
original graph, matrices, emitted GNN sections, or generated code are identical.
The one strict row, `roundtrip_strict_minimal`, is a deliberately tiny reversible
fixture used to show what the strict invariant tier requires.

The previous DRIFT targets, `cli_tool` and `notebook_module`, now begin with
source roles and roundtrip as `ROLE_PRESERVED`. They remain useful controls
because the freshness gate rejects zero-origin control-positive rows before they
can be counted as role-preserved evidence.

## Release Boundaries

Do claim:

- artifact generation for supported Python repositories;
- reproducible fixture-level roundtrip measurement;
- current test, coverage, lint, and type-check status from `METRICS.yaml`;
- role-population preservation on the native 25-target ledger; and
- explicit separation between role preservation and strict structural fidelity.

Do not claim:

- semantic equivalence for arbitrary source repositories;
- reference-device validation for runtime behavior;
- calibrated probabilities for emitted A/B/C/D matrices;
- full Rust backend parity; or
- held-out generalization beyond the measured fixture families.

## Remaining Work

| Gap | Why it matters | Required evidence |
| --- | --- | --- |
| Held-out repositories | Current fixtures were visible during rule design | Locked corpus, rerun ledger, report role and structural scores |
| Strict structural roundtrip | Current strict success is limited to the minimal reversible subset | Reduce node/edge, matrix, section, and generated-code deltas on ordinary fixtures |
| Dynamic enrichment documentation | Users need a repeatable trace-collection workflow | End-to-end tutorial with trace fixture and expected outputs |
| Rust backend parity | Current Rust path accelerates selected routines only | Stage-by-stage parity matrix and tests |
| Probability calibration | Matrix defaults are structurally valid but not empirical estimates | Gold corpus, reviewer labels, calibration report |
| Cross-language roundtrip ledger | JS/TS evidence is separate from the Python native ledger | Native multi-language rows or clearly separate claims |
