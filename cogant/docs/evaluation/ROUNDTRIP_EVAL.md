Interpretation: the current benchmark is strong in-sample evidence for
role-population preservation on the shipped Python fixtures and reductions, but it is not a
strict graph-isomorphism result. `ROLE_PRESERVED` means the semantic-role
multiset survives the forward -> reverse -> forward loop at the public threshold.
Strict structural success additionally requires zero node/edge deltas, preserved
edge-kind counts, matrix value preservation, GNN-section preservation, and
generated-code success. `roundtrip_strict_minimal` clears that stricter bar
because it is a deliberately minimal reversible subset; ordinary application
fixtures remain role-preserved but not strictly graph-isomorphic.

The previous `DRIFT` rows (`cli_tool`, `notebook_module`) now carry source
HIDDEN_STATE, OBSERVATION, and ACTION roles and roundtrip as `ROLE_PRESERVED`.
The freshness gate still rejects zero-origin control-positive rows, so generated
scaffolding cannot by itself launder a row into role-preserved evidence.

## Method

The current ledger is generated by:

```bash
cd cogant
uv run python ../tools/regenerate_roundtrip_ledger.py
uv run python ../tools/regenerate_metrics.py
```

For each locally available Python fixture under `examples/zoo/`,
`examples/control_positive/`, and `examples/real_world/`, the generator runs
`verify_repo_roundtrip`:

```text
source repo -> forward COGANT bundle -> reverse synthesized package -> forward COGANT bundle
```

Each row records:

- `roundtrip_status`
- `role_preservation_score`
- `structurally_isomorphic`
- matrix and structural scores
- generated-code status
- original and synthesized HIDDEN_STATE / OBSERVATION / ACTION counts
- source file count, LOC, graph node count, edge count, and elapsed seconds

`tools/check_metrics_fresh.py` re-derives the status distribution and native
score aggregates from `roundtrip_results.jsonl`. If `METRICS.yaml` claims richer
role-preservation counts than the JSONL can support, the freshness gate fails.

## Scope Limits

- The benchmark is in-sample: the rules and reverse synthesizer have been
  developed against these fixture families.
- JS-only fixtures are not in this Python-front-end ledger. Cross-language
  evidence is documented separately in [CROSS_LANG_ROUNDTRIP.md](CROSS_LANG_ROUNDTRIP.md).
- Role preservation is not semantic equivalence of arbitrary source programs.
  It is a measured invariant over emitted semantic-role populations plus the
  stricter invariant ledger fields listed above.
