## Configuration and precedence

- Optional YAML settings for the pipeline are loaded with [`ConfigLoader.load_from_yaml`](https://github.com/ActiveInferenceInstitute/COGANT/blob/main/cogant/py/cogant/config/loaders.py); the repo ships an example [`cogant.yaml`](https://github.com/ActiveInferenceInstitute/COGANT/blob/main/cogant/cogant.yaml) at the package root.
- CLI / `PipelineRunner` accept `--config` / `PipelineConfig` fields that point at that file; see [`SPEC` § Configuration](../reference/README.md) for the shape of merged settings.
- **Rule execution order** is defined in code: concrete [`TranslationRule`](https://github.com/ActiveInferenceInstitute/COGANT/blob/main/cogant/py/cogant/translate/rules/) classes are registered on [`TranslationEngine`](https://github.com/ActiveInferenceInstitute/COGANT/blob/main/cogant/py/cogant/translate/engine.py) in a single ordered list. There is no separate YAML-driven rule-priority layer; adding or reordering rules requires Python changes or an integration wrapper around the engine.
- **Conflict resolution** uses the current evidence contract: overlapping mappings are compared by rule priority and confidence, the selected mapping is recorded in the rule-evidence trace, and downstream dashboards report the conflict outcome instead of silently collapsing competing roles.
- **Downstream shape:** After translation, mappings are stored on the pipeline bundle as **`_semantic_mappings`** (dict: id → mapping object) and consumed by the **export** stage and [`GNNPackageBuilder`](https://github.com/ActiveInferenceInstitute/COGANT/blob/main/cogant/py/cogant/gnn/package.py). See [API guide § Export stage and GNN package](../api/README.md).
