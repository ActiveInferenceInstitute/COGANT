# Learning Paths

> Opinionated reading orders through the COGANT documentation, curated by
> audience. Each path is a sequence of existing pages chosen to match a
> specific goal — "I want to run my first scan", "I want to embed COGANT
> in my service", "I want to defend the theory in a review", etc. Pick the
> path that matches your situation and follow it end to end.

## Contents

| Path | Audience | Estimated time |
|------|----------|----------------|
| [New User](new-user.md) | Never heard of COGANT; want to install it and analyze a repo | ~45 min reading, ~30 min hands-on |
| [API Consumer](api-consumer.md) | Wants to embed COGANT in a Python service or pipeline | ~90 min reading, ~2 hr hands-on |
| [Theory Reader](theory-reader.md) | Researchers and reviewers who want the "why" before the "how" | ~3 hr reading, optional hands-on |
| [Plugin Author](plugin-author.md) | Adding languages, rules, validators, or exporters | ~2 hr reading, ~half day hands-on |
| [Contributor](contributor.md) | Fixing bugs, adding features, or landing research results in the repo itself | ~2 hr reading, open-ended hands-on |

## How to use a learning path

Each path is a curated sequence of links into the rest of the documentation;
the path itself contains almost no new content. Read the path top to bottom,
follow the links as you go, and come back here when you finish a section.
The paths are designed so that you can abandon one partway through and
still walk away with a complete, self-consistent mental model.

## Choosing a path

If you are not sure which path to follow, read [New User](new-user.md)
first. It is the shortest path and it routes you to any of the others
once you know enough to choose.
