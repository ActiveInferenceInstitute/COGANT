# Abstract

**COGANT** (Codebase-to-GNN Translation) converts software repositories into structured Active Inference artifacts expressed in the Active Inference Institute's **Generalized Notation Notation** (GNN)[^gnn-note]. It occupies the space between classical program analysis, which already reasons over graphs such as call graphs and code property graphs [@yamaguchi2014modeling], and data-driven methods that expect tensors, consistent node and edge vocabularies, and exportable training bundles [@allamanis2018survey; @wu2020comprehensive; @scarselli2009graph].

[^gnn-note]: Throughout this manuscript **GNN** refers to the Active Inference Institute's **Generalized Notation Notation** ([repository](https://github.com/ActiveInferenceInstitute/GeneralizedNotationNotation)), a structured notation for state-space and process models --- *not* graph neural networks. Downstream consumers, including graph neural network training pipelines, can ingest the emitted artifacts, but COGANT itself produces Generalized Notation Notation bundles.

The design centres on a **program graph IR** in which nodes denote entities (functions, classes, variables, modules) and edges denote relations (calls, reads, writes, imports, containment), with every assertion carrying **confidence** and **provenance**. A **fixpoint translation engine** applies **19 declarative rules** (five structural, five semantic, two control, four behavioural, three resilience) that project nodes onto **seven core Active Inference semantic roles** --- HIDDEN_STATE, OBSERVATION, ACTION, POLICY, PREFERENCE, CONSTRAINT, and CONTEXT --- iterating until no new mappings emerge and resolving conflicts by priority and confidence. Each GNN bundle carries a **Markov blanket partition** computed in $O(V+E)$ over the program graph. A **reverse synthesizer** (`cogant.reverse`) takes any emitted GNN bundle back to a runnable Python package; the forward-reverse-forward round-trip preserves the Active Inference role distribution with a measured isomorphism score $\geq 0.7$ on the shipped control-positive fixtures (see §9 and `_rnd/ISOMORPHISM_THEOREM.md`). The v0.1.0 Python front end parses **Python** directly via the standard-library `ast` module and exposes **JavaScript and TypeScript** parsers through `tree-sitter`. The shipped test suite contains **1072 passing tests** (with 56 conditional skips); the full test matrix runs on Python 3.11, 3.12, and 3.13. This manuscript, parallel to the developer documentation under [`../cogant/docs/`](../cogant/docs/), states the theory of the program graph IR, the practice of running the pipeline, and the extension points (parsers, rules, validators, exporters) through which COGANT grows.
