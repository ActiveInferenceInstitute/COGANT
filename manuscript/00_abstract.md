# Abstract {#sec:00-abstract}

**COGANT** (Codebase-to-GNN Translation) converts software repositories into structured Active Inference artifacts expressed in the Active Inference Institute's **Generalized Notation Notation** (GNN)[^gnn-note]. The system is best understood as an **evidence compiler**: it propagates reviewable program facts through a finite rule pipeline, in the tradition of fixpoint program analysis [@kildall1973unified; @cousot1977abstract], and emits graph, matrix, provenance, visualization, and roundtrip artifacts rather than a single opaque embedding or proof. It sits between classical program analysis, which already reasons over graphs such as call graphs and code property graphs [@yamaguchi2014modeling], and data-driven methods that expect tensors, consistent node and edge vocabularies, and exportable training bundles [@allamanis2018survey; @wu2020comprehensive; @scarselli2009graph].

**Materials and methods.** The design centres on a **program graph intermediate representation (IR)** whose nodes and edges carry **confidence** and **provenance**. A **fixpoint translation engine** applies **{{TRANSLATION_RULES}} declarative rules** (structural, semantic, control, behavioural, and resilience families) to map nodes onto **{{ACTIVE_INF_ROLE_COUNT}}** Active Inference mapping kinds in `cogant.schemas.semantic.MappingKind` (see @sec:02-01-program-graph-and-formal-foundations for IR layers, definitions, and schema detail). Conflict resolution uses priority-and-score ordering, and the rule-evidence trace records matched nodes, confidence components, conflicts, and reviewer outcomes in a PROV-compatible spirit [@moreau2013prov]. Each emitted bundle includes a **seed-induced structural partition** using Markov-blanket role vocabulary in $O(V+E)$ time; this is an interface decomposition over the recovered program graph, not a probabilistic conditional-independence proof. A **reverse synthesizer** (`cogant.reverse`) reconstructs a runnable Python package from an emitted GNN bundle, closing a forward-reverse-forward evaluation loop. The materials are the version-controlled fixtures, optional remote targets declared in `run_all.json`, regenerated `METRICS.yaml`, `metrics.json`, run manifests, figure sidecars, claim ledgers, and software-archive metadata described in @sec:06-experimental-setup and @sec:07-reproducibility [@smith2016softwareCitationPrinciples; @vanDeSandt2019zenodoSoftwareCitations]; no headline number is introduced from an unregistered prose-only calculation.

**Evidence (round-trip).** The v{{VERSION}} roundtrip ledger reports `role_preservation_score` as the per-role mean of $\min/\max$ multiset similarity over the union of original and synthesized roles (@sec:S01-appendix-roundtrip-epsilon) — a macro-averaged per-role min/max multiset similarity that penalises **both** dropped and hallucinated roles [@wu2018weightedMinhash; @moulton2018jaccardProbability] — computed live per run and kept strictly separate from `roundtrip_status`. The **{{TOTAL_TARGETS}}**-target regression corpus[^fixture-taxonomy] (`roundtrip_results.jsonl`) is a **current native ledger** regenerated by `tools/regenerate_roundtrip_ledger.py`, which runs forward→reverse→forward on every shipped fixture (zoo, control-positive, and real-world reductions) and records each target's native `role_preservation_score`, `roundtrip_status`, per-role multiset counts, and graph size. On this ledger `METRICS.yaml` reports **{{ROLE_PRESERVED_COUNT}}** role-preserved targets (`s_role` $\geq$ {{THRESHOLD_ROLE_PRESERVED}}), a strict structural-isomorphism count of **{{STRICT_ISOMORPHISM_COUNT}}**, and **{{DRIFT_COUNT}}** drift targets, with native mean/median `role_preservation_score` of **{{MEAN_ROLE_PRESERVATION_SCORE}}** / **{{MEDIAN_ROLE_PRESERVATION_SCORE}}** and an explicit score source of `{{ROLE_PRESERVATION_SCORE_SOURCE}}`. Strict structural isomorphism additionally requires zero node/edge deltas, preserved edge-kind counts, matrix shape/value preservation, GNN-section preservation, and generated-code compile success; the strict subset contains **{{STRICT_ISOMORPHISM_COUNT}}** of {{TOTAL_TARGETS}} targets. The strict row is a deliberately minimal reversible fixture, not evidence that ordinary application fixtures are graph-isomorphic. Role preservation is therefore a symmetric role-overlap score, **not** graph isomorphism and **not** a recall estimate. These fixtures are in-sample — the {{TRANSLATION_RULES}} rules were authored with them visible — so the scores describe a deterministic artifact/regression corpus and upper-bound, rather than estimate, out-of-sample role recovery (see the external-validity caveat below and the empirical-software-engineering framing in @sec:06-experimental-setup) [@ralph2020empiricalStandards; @hasselbring2021benchmarking]. Per-run live roundtrip metrics, including the `calculator` fixture, are reported with structural diffs rather than a single saturating score in @sec:S01-appendix-roundtrip-epsilon.

**Scope, front ends, and quality gate.** The v{{VERSION}} pipeline uses Python's standard-library `ast` as its primary front end. **JavaScript/TypeScript** and optional services (HTTP, container, incremental `cogant analyze --incremental <git-ref>`, multi-episode agent runtime) are documented in the package documentation under `cogant/docs/` and the package `README.md`; this manuscript focuses on IR theory, practice, and extension points (parsers, rules, validators, exporters). On the canonical run, the test suite reports **{{TEST_COUNT_PASSING}}** passing tests, **{{TEST_COUNT_FAILING}}** failing tests, **{{TEST_COUNT_SKIPPED}}** skipped tests, **{{TEST_COUNT_XFAILED}}** expected `xfail`, and **{{TEST_COUNT_XPASSED}}** `xpass` case; failures are treated as release blockers rather than silently excluded from the evidence record. The same run reports **{{COVERAGE_PCT}}%** line coverage of `cogant` (`uv run pytest tests/ --cov=py/cogant`, **{{METRICS_GENERATED_AT}}**) and **{{MYPY_STRICT_ERRORS}}** `mypy --strict` findings. These gates support implementation reliability, reproducibility, and regression detection; they do not by themselves establish semantic adequacy of a translated repository. Supported interpreters: Python 3.11–3.13.

**External-validity caveat.** The {{TRANSLATION_RULES}} translation rules and the
fixture-specific keyword vocabularies were authored with the shipped
evaluation fixtures visible. There is **no held-out test split** and **no
confidence interval** on any reported aggregate; in-sample scores
*upper-bound*, and do not estimate, performance on never-tuned
repositories. Claims phrased as "the pipeline runs end-to-end on
repository X" carry no implied claim of role-recovery accuracy on
repositories outside the shipped fixture set. See @sec:08-05-threats-to-validity
for the full caveat.

[^gnn-note]: **GNN** here is Generalized Notation Notation, not graph neural networks; see the terminology block in @sec:01-introduction.

[^fixture-taxonomy]: Three nested fixture counts appear in this manuscript and are not in tension. The **six** primary benchmark fixtures tabulated in @sec:06-03-performance-and-fixture-metrics (`calculator`, `event_pipeline`, `flask_mini`, `flask_app`, `requests_lib`, `json_stdlib`) are a subset of the **{{SHIPPED_FIXTURE_COUNT}}** shipped repositories exercised end-to-end, which are in turn a subset of the **{{TOTAL_TARGETS}}** round-trip targets scored here (zoo + control-positive + real-world reductions; see @sec:S01-appendix-roundtrip-epsilon and @sec:08-05-threats-to-validity). The benchmark tables list the six; the round-trip ledger scores all {{TOTAL_TARGETS}}.
