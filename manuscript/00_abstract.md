# Abstract {#sec:00-abstract}

**COGANT** (Codebase-to-GNN Translation) converts software repositories into structured Active Inference artifacts expressed in the Active Inference Institute's **Generalized Notation Notation** (GNN)[^gnn-note]. It sits between classical program analysis, which already reasons over graphs such as call graphs and code property graphs [@yamaguchi2014modeling], and data-driven methods that expect tensors, consistent node and edge vocabularies, and exportable training bundles [@allamanis2018survey; @wu2020comprehensive; @scarselli2009graph].

**Method.** The design centres on a **program graph IR** whose nodes and edges carry **confidence** and **provenance**. A **fixpoint translation engine** applies **{{TRANSLATION_RULES}} declarative rules** (structural, semantic, control, behavioural, and resilience families) to map nodes onto **{{ACTIVE_INF_ROLE_COUNT}}** Active Inference mapping kinds in `cogant.schemas.semantic.MappingKind` (see @sec:02-01-program-graph-and-formal-foundations for IR, definitions, and schema detail). Conflict resolution uses priority-and-score ordering; each emitted bundle includes a **Markov blanket partition** in $O(V+E)$ time. A **reverse synthesizer** (`cogant.reverse`) reconstructs a runnable Python package from an emitted GNN bundle, closing a forward–reverse–forward evaluation loop.

**Evidence (round-trip).** After the v{{VERSION}} fix to POLICY/CONTEXT stub emission ([`ROUNDTRIP_IMPROVEMENT.md`](../cogant/docs/evaluation/ROUNDTRIP_IMPROVEMENT.md)), round-trip evaluation on the canonical **{{TOTAL_TARGETS}}**-target set reports **{{ISO_COUNT}}** isomorphic ($\varepsilon \geq {{THRESHOLD_ISO}}$), **{{APPROX_COUNT}}** approximate ($\varepsilon \geq {{THRESHOLD_APPROX}}$), and **{{DIV_COUNT}}** divergent targets under the thresholds recorded in `METRICS.yaml` (`evaluation.roundtrip`). On the canonical **post-wave-16** benchmark wired to that file ([`ROUNDTRIP_EVAL.md`](../cogant/docs/evaluation/ROUNDTRIP_EVAL.md)), the same injected counts classify **{{ISO_COUNT}}/{{TOTAL_TARGETS}}** targets as ISOMORPHIC, **{{APPROX_COUNT}}** as APPROXIMATE, and **{{DIV_COUNT}}** as DIVERGENT—the tier labels remain the taxonomy for regressions, older waves, and extended corpora (Appendix A).

**Scope, front ends, and quality gate.** The v{{VERSION}} pipeline uses Python's standard-library `ast` as its primary front end. **JavaScript/TypeScript** and optional services (HTTP, container, incremental `cogant analyze --incremental <git-ref>`, multi-episode agent runtime) are documented in [`cogant/docs/index.md`](../cogant/docs/index.md) and the package [`README.md`](../cogant/README.md); this manuscript focuses on IR theory, practice, and extension points (parsers, rules, validators, exporters). On the canonical run, the test suite reports **{{TEST_COUNT}}** passing tests (with skips/xfail/xpass as in `METRICS.yaml`), **{{COVERAGE_PCT}}%** line coverage of `cogant` (`uv run pytest tests/ --cov=cogant`, **{{METRICS_GENERATED_AT}}**), and **{{MYPY_STRICT_ERRORS}}** `mypy --strict` findings. Supported interpreters: Python 3.11–3.13.

[^gnn-note]: **GNN** here is Generalized Notation Notation, not graph neural networks; see the terminology block in @sec:01-introduction.
