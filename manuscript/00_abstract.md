# Abstract

**COGANT** (Codebase-to-GNN Translation) converts software repositories into structured Active Inference artifacts expressed in the Active Inference Institute's **Generalized Notation Notation** (GNN)[^gnn-note]. It sits between classical program analysis, which already reasons over graphs such as call graphs and code property graphs [@yamaguchi2014modeling], and data-driven methods that expect tensors, consistent node and edge vocabularies, and exportable training bundles [@allamanis2018survey; @wu2020comprehensive; @scarselli2009graph].

[^gnn-note]: Throughout this manuscript **GNN** refers to the Active Inference Institute's **Generalized Notation Notation** ([repository](https://github.com/ActiveInferenceInstitute/GeneralizedNotationNotation)), a structured notation for state-space and process models --- *not* graph neural networks. Downstream consumers, including graph neural network training pipelines, can ingest the emitted artifacts, but COGANT itself produces Generalized Notation Notation bundles.

The design centres on a **program graph IR** whose nodes and edges carry **confidence** and **provenance**. A **fixpoint translation engine** applies **{{TRANSLATION_RULES}} declarative rules**—organized into five structural, five semantic, two control, three behavioural, and four resilience families—to map nodes onto **{{ACTIVE_INF_ROLE_COUNT}} Active Inference semantic roles** (HIDDEN\_STATE, OBSERVATION, ACTION, POLICY, PREFERENCE, CONSTRAINT, CONTEXT). Conflict resolution follows a priority-and-score ordering; each emitted bundle includes a **Markov blanket partition** computed in $O(V+E)$ time. A **reverse synthesizer** (`cogant.reverse`) reconstructs a runnable Python package from an emitted GNN bundle, closing a forward–reverse–forward evaluation loop. After the v{{VERSION}} fix to POLICY/CONTEXT stub emission ([`ROUNDTRIP_IMPROVEMENT.md`](../cogant/docs/evaluation/ROUNDTRIP_IMPROVEMENT.md)), round-trip evaluation on the canonical {{TOTAL_TARGETS}}-target set (twelve zoo fixtures, three real-world-example fixtures, eight third-party libraries) achieves **{{ISO_COUNT}} isomorphic** ($\varepsilon \geq {{THRESHOLD_ISO}}$), **{{APPROX_COUNT}} approximate** ($\varepsilon \geq {{THRESHOLD_APPROX}}$), and **{{DIV_COUNT}} divergent** targets. Per-target tables and tier definitions are in [`ROUNDTRIP_EVAL.md`](../cogant/docs/evaluation/ROUNDTRIP_EVAL.md) and Appendix A.

The v{{VERSION}} pipeline uses Python's standard-library `ast` module as its primary front end. **JavaScript and TypeScript support is optional**: install the `cogant[multilang]` extra and an available `tree-sitter` grammar; integration tests cover a JS-grammar fallback path for `.ts` files on mixed repositories when dependencies are present. HTTP service, container image, incremental analysis (`cogant analyze --incremental <git-ref>`, with measured 19.6× no-change speedup), and multi-episode agent runtime are documented in [`cogant/docs/index.md`](../cogant/docs/index.md) and the package [`README.md`](../cogant/README.md); this manuscript covers the IR theory, running practice, and extension points (parsers, rules, validators, exporters) instead. The test suite reports **{{TEST_COUNT}} passing** tests with **{{TEST_COUNT_SKIPPED}}** skips for optional dependencies, **{{TEST_COUNT_XFAILED}}** expected `xfail`, and **{{TEST_COUNT_XPASSED}}** `xpass`; `py/cogant/` line coverage is **{{COVERAGE_PCT}}%** on the canonical run (generated **{{METRICS_GENERATED_AT}}**). Supported interpreters: Python 3.11–3.13.
