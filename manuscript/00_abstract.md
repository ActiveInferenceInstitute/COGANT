# Abstract {#sec:00-abstract}

**COGANT** (Codebase-to-GNN Translation) converts software repositories into structured Active Inference artifacts expressed in the Active Inference Institute's **Generalized Notation Notation** (GNN)[^gnn-note]. The system is best understood as an **evidence compiler**: it propagates reviewable program facts through a finite rule pipeline, in the tradition of fixpoint program analysis [@kildall1973unified; @cousot1977abstract], and emits graph, matrix, provenance, visualization, and roundtrip artifacts rather than a single opaque embedding or proof. It sits between classical program analysis, which already reasons over graphs such as call graphs and code property graphs [@yamaguchi2014modeling], and data-driven methods that expect tensors, consistent node and edge vocabularies, and exportable training bundles [@allamanis2018survey; @wu2020comprehensive; @scarselli2009graph].

**Materials and methods.** The design centres on a **program graph intermediate representation (IR)** whose nodes and edges carry **confidence** and **provenance**. A **fixpoint translation engine** applies **{{TRANSLATION_RULES}} declarative rules** (structural, semantic, control, behavioural, and resilience families) to map nodes onto **{{ACTIVE_INF_ROLE_COUNT}}** Active Inference mapping kinds in `cogant.schemas.semantic.MappingKind` (see @sec:02-01-program-graph-and-formal-foundations for IR layers, definitions, and schema detail). Conflict resolution uses priority-and-score ordering, and the rule-evidence trace records matched nodes, confidence components, conflicts, and reviewer outcomes in a PROV-compatible spirit [@moreau2013prov]. Each emitted bundle includes a **Markov blanket partition** in $O(V+E)$ time. A **reverse synthesizer** (`cogant.reverse`) reconstructs a runnable Python package from an emitted GNN bundle, closing a forward-reverse-forward evaluation loop. The materials are the version-controlled fixtures, optional remote targets declared in `run_all.json`, regenerated `METRICS.yaml`, `metrics.json`, run manifests, figure sidecars, and claim ledgers described in @sec:06-experimental-setup and @sec:07-reproducibility; no headline number is introduced from an unregistered prose-only calculation.

**Evidence (round-trip).** The v{{VERSION}} roundtrip ledger reports `role_preservation_score` as the per-role mean of $\min/\max$ multiset similarity (@sec:S01-appendix-roundtrip-epsilon) — a measure that penalises **both** dropped and hallucinated roles — computed live per run and kept strictly separate from `roundtrip_status`. The historical **{{TOTAL_TARGETS}}**-target benchmark (`roundtrip_results.jsonl`) is a **legacy v0.5 ε corpus retained only as a role-recall regression set**; it is no longer relabelled as a fresh v0.6 result, so on it `METRICS.yaml` reports **{{ROLE_PRESERVED_COUNT}}** fresh-v0.6 role-preserved targets and **{{STRICT_ISOMORPHISM_COUNT}}** strict structurally isomorphic targets (legacy rows are tagged `STALE_LEGACY`, not counted as preserved). Aggregate score provenance is reported explicitly as `{{ROLE_PRESERVATION_SCORE_SOURCE}}`; when only legacy ε rows are present, native v0.6 mean/median/min/max score fields render as `N/A` rather than reusing ε as if it were the new metric. Strict structural isomorphism additionally requires zero node/edge deltas, preserved edge-kind counts, matrix shape/value preservation, GNN-section preservation, and generated-code compile success. Per-run live roundtrip metrics (e.g. the `calculator` fixture) are reported with their structural diffs rather than a single saturating score ([`ROUNDTRIP_EVAL.md`](../cogant/docs/evaluation/ROUNDTRIP_EVAL.md)).

**Scope, front ends, and quality gate.** The v{{VERSION}} pipeline uses Python's standard-library `ast` as its primary front end. **JavaScript/TypeScript** and optional services (HTTP, container, incremental `cogant analyze --incremental <git-ref>`, multi-episode agent runtime) are documented in [`cogant/docs/index.md`](../cogant/docs/index.md) and the package [`README.md`](../cogant/README.md); this manuscript focuses on IR theory, practice, and extension points (parsers, rules, validators, exporters). On the canonical run, the test suite reports **{{TEST_COUNT_PASSING}}** passing tests, **{{TEST_COUNT_FAILING}}** failing tests, **{{TEST_COUNT_SKIPPED}}** skipped tests, **{{TEST_COUNT_XFAILED}}** expected `xfail`, and **{{TEST_COUNT_XPASSED}}** `xpass` case; failures are treated as release blockers rather than silently excluded from the evidence record. The same run reports **{{COVERAGE_PCT}}%** line coverage of `cogant` (`uv run pytest tests/ --cov=py/cogant`, **{{METRICS_GENERATED_AT}}**) and **{{MYPY_STRICT_ERRORS}}** `mypy --strict` findings. Supported interpreters: Python 3.11–3.13.

**External-validity caveat.** The {{TRANSLATION_RULES}} translation rules and the
fixture-specific keyword vocabularies were authored with the shipped
evaluation fixtures visible. There is **no held-out test split** and **no
confidence interval** on any reported aggregate; in-sample scores
*upper-bound*, and do not estimate, performance on never-tuned
repositories. Claims phrased as "the pipeline runs end-to-end on
repository X" carry no implied claim of role-recovery accuracy on
repositories outside the shipped fixture set. See @sec:08-05-threats-to-validity
for the full caveat.

[^gnn-note]: **GNN** here is Generalized Notation Notation, not graph neural networks; see the terminology block in @sec:01-introduction.
